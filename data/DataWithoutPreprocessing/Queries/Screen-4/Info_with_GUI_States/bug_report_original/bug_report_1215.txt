Crash when entering too large of a number in manual entry dialog

General information
App version: 0.70 and 0.7.1.1-dev
App source: F-Droid and GitHub master
Android Version: 10 (Samsung S9 and x86 emulator)

Steps to reproduce
TOTP Crash
1. Open app to the main list of 2FA codes.
2. Tap the add FAB and select "Enter details".
3. Select TOTP for the Type field.
4. Enter something into the Label and Secret fields.
5. Open the Advanced Options overflow, and enter a number larger than Integer.MAX_VALUE into either the Period or Digits fields.
HOTP Crash
1. Open app to the main list of 2FA codes.
2. Tap the add FAB and select "Enter details".
3. Select HOTP for the Type field.
4. Enter something into the Label and Secret fields.
5. Enter a number larger than Long.MAX_VALUE into the Counter field.
6. Tap the Save button.

Expected result

What is expected?
In the manual details entry dialog, entering a number in the Period or Digits fields larger than Integer.MAX_VALUE or entering a number in the Counter field larger than Long.MAX_VALUE should disable the Save button.

What does happen instead?
The application crashes. This would probably never occur during normal usage, but it's an easy fix so I'll put a PR up.
